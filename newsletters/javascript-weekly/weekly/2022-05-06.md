## Top Stories

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60/uba1eccha06rxyixtymv.jpg)

[How Partytown Eliminates Website Bloat From Third-Party Scripts](https://www.smashingmagazine.com/2022/04/partytown-eliminates-website-bloat-third-party-apps/) — As well as being the name of my house, [Partytown](https://partytown.builder.io/) is also a library for moving the execution of scripts into the background via [Web Workers](https://developer.mozilla.org/en-US/docs/Web/API/Web_Workers_API/Using_web_workers) thus freeing up your main thread. Why does that matter? Let Steve tell you.

Steve Sewell

[Ryan Dahl on 'JavaScript Containers'](https://tinyclouds.org/javascript_containers) — Ryan, originally known for Node and now [Deno](https://deno.land/), thinks about JavaScript as being a universal scripting language and how the JS sandbox acts as a sort of high level version of the traditional Linux container and will only become more important in the next few years.

Ryan Dahl

[Why I Don’t Miss React](https://www.jackfranklin.co.uk/blog/working-with-react-and-the-web-platform/) — *Not* a call to drop React, a declaration of it being ‘dead’, or that all frameworks are useless, but the author went from using React every day to not using it at all, and had a more pleasant time than he was expecting.

As usual, they have [a release video](https://www.youtube.com/watch?v=zELYw2qEUjI) showing it off or you can hit [the playground](https://playground.babylonjs.com/) to get playing with a basic example in seconds.

**IN BRIEF:**

- [TodoMVC](https://todomvc.com/) is a long standing project showing how a ToDo app can be built in numerous frameworks. But what about *no* framework? Marc Grabanski [shows us TodoMVC in vanilla JS](https://github.com/1Marc/todomvc-vanillajs-2022) in 2022.
- I'm seeing a *lot* of library and tool releases explicitly dropping Node 12 support in their latest versions, so if you're still using Node 12 anywhere, upgrade ASAP (or, at least, take a lot of care when updating dependencies).
- [Node 18.1.0](https://nodejs.org/en/blog/release/v18.1.0/) has been released and includes [a new test running feature](https://nodejs.org/api/test.html#running-tests-from-the-command-line) on the CLI: `node --test` *(Be sure to see [Node Weekly](https://nodeweekly.com/) for more on Node.)*

## 🐍 We're Not *Python Weekly* But We've Still Got Some Python Stuff for You Anyway

[JSPyBridge 1.0: A Bridge Between Node and Python](https://github.com/extremeheat/JSPyBridge) — A work in progress but offers the ability to go either way - run Python from Node, or run Node from Python ‘with a native feel.’ *“Unlike other bridges, you may notice you’re not just writing Python code in JavaScript, or vice-versa. You can operate on objects on the other side of the bridge as if the objects existed on your side.”*

extremeheat

[Pyscript: Run Python in Your HTML](https://pyscript.net/) — A framework for creating Python apps in the browser by way of HTML. You get bi-directional communication with JavaScript, too. Unsurprisingly, WebAssembly is what does the heavy lifting and makes this all possible. [More background here.](https://www.anaconda.com/blog/pyscript-python-in-the-browser)

Anaconda Inc.
