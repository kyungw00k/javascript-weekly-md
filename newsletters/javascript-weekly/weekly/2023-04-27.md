[Babylon.js 6.0: The Powerful Web-Based 3D Framework](https://babylonjs.medium.com/announcing-babylon-js-6-0-dcb5f1662e3a) — Babylon.js remains one of the world’s leading WebGL-based graphics engines with a visual scene builder and best-in-class physically-based rendering. v6.0 includes [a new physics plugin](https://babylonjs.medium.com/physics-v2-overview-ed36039ce1e7) with plenty of [docs](https://doc.babylonjs.com/features/featuresDeepDive/physics/rigidBodies) and [demos](https://theplayroom.babylonjs.com/), [fluid rendering](https://doc.babylonjs.com/features/featuresDeepDive/particles/fluid_renderer), major improvements to how [reflections](https://doc.babylonjs.com/features/featuresDeepDive/postProcesses/SSRRenderingPipeline) are handled, [screen reader support](https://doc.babylonjs.com/toolsAndResources/accessibility/screenReaders), and more. This is a *huge* release of a significant project in the JavaScript ecosystem and we can’t do it justice here but there’s [even more on the official homepage.](https://www.babylonjs.com/)

**IN BRIEF:**

- 🕹 Turns out even Nintendo uses JavaScript. [A developer discovered](https://twitter.com/JasperRLZ/status/1648046875675856897) that 2015's *Mario vs. Donkey Kong: Tipping Stars* for the Wii U and 3DS was written with HTML and JavaScript under the hood and managed to build a shim to get it running in a normal browser.
- [Chrome 113's DevTools](https://developer.chrome.com/blog/new-in-devtools-113/) will allow you to *override network response headers*, including CORS headers. It also offers Nuxt, Vite, and Rollup debugging improvements.
- 🐦 ["Switching to rspack from webpack+babel has reduced our build times by 85%."](https://twitter.com/shawnrmcknight/status/1651275500118065175)
- We covered Node.js 20's release last week, but if you want to dig deeper into the new (experimental) permissions model, [here are the full docs.](https://nodejs.org/api/permissions.html#permissions)
- In a recent [bug report on the React repo](https://github.com/facebook/react/issues/26704#issuecomment-1522044060) Dan Abramov noted that if you use [Preact Signals](https://preactjs.com/guide/v10/signals/) with React, you forfeit any guarantee that React will work correctly and that *"if React was a piece of hardware, this is exactly the kind of thing that voids the warranty."*
- A sneak peek of [what's coming up in Angular 16.](https://itnext.io/angular-16-is-huge-67288a3ff58b)
