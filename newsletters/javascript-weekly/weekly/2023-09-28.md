# JavaScript Weekly Issue 656: September 28, 2023

## Top Stories

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/o2or5d4yoqle5ghfr2tn.jpg)

[The Saga of Google's Closure Compiler](https://effectivetypescript.com/2023/09/27/closure-compiler/) — Dan looks back at Google’s [*Closure Compiler*](https://developers.google.com/closure/compiler/), a JavaScript transpiler Google built in 2004 and used most heavily in the pre-TypeScript era to reduce the size of JavaScript files, check types, and otherwise handle common pitfalls. A neat bit of JavaScript history.

Dan Vanderkam

[Speeding up the JavaScript Ecosystem: Polyfills Gone Rogue](https://marvinh.dev/blog/speeding-up-javascript-ecosystem-part-6/) — Marvin's mission to speed up popular libraries continues. First [blogging on the topic](https://marvinh.dev/blog/speeding-up-javascript-ecosystem/) last year, now he’s looking at the role polyfills play in making packages large and inefficient. There are two sides to every story, though, as [this Hacker News thread found](https://news.ycombinator.com/item?id=37604373).

Marvin Hagemeister

![](https://copm.s3.amazonaws.com/71aadf41.jpeg)

[Building Cross-Platform Desktop Apps with Electron](https://frontendmasters.com/courses/electron-v3/) — In this detailed video course Steve Kinney shares how to build robust, user-friendly Electron-based desktop apps, covering key concepts like main and renderer processes, inter-process communication, security, and more.

**⚡️ IN BRIEF:**

- [🐦 TC39 met](https://twitter.com/robpalmer2/status/1707331795124179110) earlier today and [advanced several proposals](https://twitter.com/robpalmer2/status/1707331777440919595) including [RegExp escaping](https://github.com/tc39/proposal-regex-escaping) and [resizable ArrayBuffers](https://github.com/tc39/proposal-resizablearraybuffer) (and nixed two, including [Symbol.thenable](https://github.com/tc39/proposal-symbol-thenable)).
- If possible, please [take the *State of HTML 2023* survey.](https://survey.devographics.com/en-US/survey/state-of-html/2023) Your answers could help determine what features browsers focus on next! And if you don't believe me, [let Lea Verou convince you](https://lea.verou.me/blog/2023/state-of-html-2023/).
- [Verticalize](https://github.com/laurentpayot/verticalize) introduces a vertical pipe-like function to JavaScript to help you chain operations together in a more visually appealing way, but.. it's certainly a matter of taste.
- The [Preact](https://preactjs.com/) team has [an update on Preact Signals](https://preactjs.com/blog/signal-boosting/), their take on reactive programming.
- Engineers from Vercel and Cloudflare have [published a draft spec](https://sockets-api.proposal.wintercg.org/) for a Sockets API for non-browser runtimes. Here's [why this all matters.](https://blog.cloudflare.com/socket-api-works-javascript-runtimes-wintercg-polyfill-connect/)
- 📱 [Strada](https://dev.37signals.com/announcing-strada/) is a new library from 37signals that bridges the gap between server-rendered HTML and native controls in hybrid mobile apps. Here's [how it looks in practice](https://masilotti.com/strada-launch/).
- Publishing packages [with npm provenance is now generally available](https://github.blog/changelog/2023-09-26-npm-provenance-general-availability/).
- [Cloudflare Workers](https://developers.cloudflare.com/workers/) is a popular serverless platform for running JavaScript functions at the edge and it now has [an all new pricing model](https://blog.cloudflare.com/workers-pricing-scale-to-zero/) that's certainly a lot more appealing.
