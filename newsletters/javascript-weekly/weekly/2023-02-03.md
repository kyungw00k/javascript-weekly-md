## Top Stories

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/lxrxytcdhrnvp2tlpu3n.jpg)

[You’ve Got Options for Removing Event Listeners](https://www.macarthur.me/posts/options-for-removing-event-listeners) — Unnecessary event listeners can cause all sorts of odd problems so it’s good to clean them up when you don’t need them anymore. How? There are several approaches and Alex looks at their pros and cons. (`once` is a good one to consider if your use case supports it as it’s ‘set and forget.’)

Alex MacArthur

[Updates from the 94th TC39 meeting](https://dev.to/hemanth/updates-from-the-94th-tc39-meeting-48mb) — The TC39 committee that works on the ECMAScript standard met last week and progressed a few language proposals with [Change Array by Copy](https://github.com/tc39/proposal-change-array-by-copy), [Intl.NumberFormat v3](https://github.com/tc39/proposal-intl-numberformat-v3) and [Symbols as WeakMap Keys](https://github.com/tc39/proposal-symbols-as-weakmap-keys) making it to stage 4. There’s also an interesting example of [a *downgrade* to stage 2 for import assertions](https://docs.google.com/presentation/d/1HbWhyo4tSnpv4vMZqCa2YQvi_mKdpDi4JWWBtSUQqQY/edit#slide=id.p).

Hemanth HM

[The Future of *Create React App* and Why It Exists](https://github.com/reactjs/reactjs.org/pull/5487#issuecomment-1409720741) — An extensive write up from Dan Abramov on the state of *Create React App*, a route to take it forwards, and how he sees React as a library working within an ecosystem of frameworks.
