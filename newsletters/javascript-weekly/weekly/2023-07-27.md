# JavaScript Weekly Issue 649: July 27, 2023

## Top Stories

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/vgrhvmxknfbuarmij9da.jpg)

[Hono + Htmx + Cloudflare: A New Stack?](https://blog.yusu.ke/hono-htmx-cloudflare/) — A lot of people seem to be jumping on [htmx](https://htmx.org/) lately as an alternative to the complexity of frameworks like React, but what if you want to keep using JSX? [Hono](https://hono.dev/) is a (vaguely Express-like) Web framework targeting edge function use cases and includes middleware for rendering JSX. Yusuke gives a quick example of how all this can come together with Cloudflare Workers and D1 for a simpler, full-stack JS experience.

Yusuke Wada

[Shrinking VS Code with Name Mangling](https://code.visualstudio.com/blogs/2023/07/20/mangling-vscode) — There’s a fair bit of JavaScript in VS Code but the team has managed to reduce the size of the shipped code by almost 4MB *without* actually doing any deleting or refactoring thanks to a new ‘name mangling’ build step. This is a great read on how the team approached things and got it working.

Matt Bierner

![](https://copm.s3.amazonaws.com/6e5f101b.png)

[New Course: The Hard Parts of UI Development](https://frontendmasters.com/courses/hard-parts-ui-dev/?utm_source=email&utm_medium=javascriptweekly&utm_content=hardpartsuidev) — Develop an under-the-hood knowledge of UI development by learning techniques such as data binding, UI composition, templating, virtual DOM and its reconciliation, and hooks, all from scratch!

Frontend Masters sponsor

[Microsoft TypeChat: An Approach for Type-Safe LLM Responses](https://microsoft.github.io/TypeChat/blog/introducing-typechat/) — A new project with some prominent names attached (the designer of C# and PM for TypeScript, for starters) that demonstrates the excitement within MS for large language models (LLMs). *TypeChat* aims to work around the problem of LLMs outputting hard-to-parse natural language and to direct such output into a predictable, *typed* form.

Hejlsberg, Lucco, Rosenwasser et al.

## 🙈 Taking things too far..

We'll support anyone who's keen to reduce the amount of needless JavaScript that goes over the wire (👋 [Astro](https://astro.build/) or [Qwik](https://qwik.builder.io/)) but sometimes, just sometimes, you can go too far with the endeavor to almost comedic effect:

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/lbs06qqqzkpdrtbtodig.jpg)

😱 [Implementing Tic Tac Toe with 170MB of HTML](https://portswigger.net/blog/tic-tac-toe-in-html) — Unsurprisingly, the demo (which we’ve *not* linked to here) brought my browser to a crashing halt, but it’s fun to see a developer take on the challenge of using Chrome’s support for popovers and the state of said popovers along with a huge pile of HTML to handle a game of Tic Tac Toe. Don’t repeat this in production, folks..

Gareth Heyes
