[Idle Until Urgent: A Performance Strategy](https://philipwalton.com/articles/idle-until-urgent/) — When analyzing his site’s ‘first input delay’, Phil noticed all the JavaScript functions being run as the page loaded added up to a significant amount of time. His solution? Only run code in idle periods or when it’s truly, *urgently* required. Here’s a look at how that works in practice.
