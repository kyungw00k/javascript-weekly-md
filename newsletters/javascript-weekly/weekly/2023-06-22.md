[UnsuckJS: Comparing Lightweight JavaScript Options](https://unsuckjs.com/) — *"No build tools, no compilers, and no hassle."* It’s a table of frontend JavaScript libraries on a simple page – that’s it! But it’s a handy table, letting you see the relative popularity, size, and latest versions of libraries as diverse as Preact, bau, htmx, Hyperapp, and Mithril.

[Svelte 4 Released](https://svelte.dev/blog/svelte-4) — Four years on from [Svelte 3.0](https://svelte.dev/blog/svelte-3-rethinking-reactivity) comes the latest major release of the popular compile-time framework that isn’t afraid to do things its own way. If you’re new to it, [hit the interactive tutorial](https://learn.svelte.dev/tutorial/welcome-to-svelte) to get a feel for things, or if you’re already a user, enjoy the [v3 to v4 migration guide](https://svelte.dev/docs/v4-migration-guide), [overhauled site](https://svelte.dev/), and extra performance on offer.

Rich Harris and the Svelte Team

[A Look at TypeScript 5.2's New Keyword: `using`](https://www.totaltypescript.com/typescript-5-2-new-keyword-using) — `using` brings something akin to Python’s `with` context management into TypeScript by providing a way to automatically run a `Symbol.dispose` function when an object leaves scope. You could use this to shut down database connections, close file handles, etc.

Matt Pocock

If you're not a TS fan, the idea behind `using` is [also an ECMAScript proposal](https://github.com/tc39/proposal-explicit-resource-management) (currently at stage 3) called *Explicit Resource Management* and there's more detail to enjoy in [their examples.](https://github.com/tc39/proposal-explicit-resource-management#motivations)

[The 'Getting Started with AI' Stack](https://a16z.com/2023/06/21/the-getting-started-with-ai-stack-for-javascript/) — Andreessen Horowitz (a.k.a. a16z) is a well known VC firm, but they also have an engineering team of their own that has come up with a [“getting started with AI” template](https://github.com/a16z-infra/ai-getting-started) for JavaScript devs who want to play with modern ML technologies without too much thinking about tooling.

Li, Li, and Casado (Andreessen Horowitz)
