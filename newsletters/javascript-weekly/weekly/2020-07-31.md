# JavaScript Weekly Issue 499: July 31, 2020

## Top Stories

#499 — July 31, 2020

[Unsubscribe](https://javascriptweekly.com/leave/*%7CUID%7C*) | [Read on the Web](https://javascriptweekly.com/issues/499)

This *felt* like a really quiet week in the JavaScript space, so I was a bit worried about finding enough cool things to include.. but we did 😄 Turns out if you scratch around a bit, there's a lot of great work going on – be sure to take a good skim through. Oh, and next week is.. ISSUE 500! 🎉

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60/v1596208279/qisn8o4tr8p5yr5xtumk.png)

[I Created The Exact Same App in React and Vue And..](https://sunilsandhu.com/posts/i-created-the-exact-same-app-in-react-and-vue-2020-edition) — Two years ago, the author built the same app in React and Vue to see the differences between the two. Since then, both have changed, so he’s back with a 2020 edition bringing hooks and Vue 3 into play.

Sunil Sandhu

[Speculation in JavaScriptCore](https://webkit.org/blog/10308/speculation-in-javascriptcore/) — A long, technical post about *speculative compilation* and how it makes a dynamic language like JavaScript a lot faster than it might otherwise be (or *was*, pre V8!) You’ll need some experience with how compilation works to get much out of this, but if you want to know how modern JavaScript VMs can run JavaScript so quickly, it’s an epic resource.

Filip Pizlo (WebKit)

![](https://copm.s3.amazonaws.com/83b0761f.png)

[Build Full-Stack Serverless Web Apps with This JS Framework](https://www.webiny.com/) — If you are looking to build full-stack serverless web apps, then Webiny is the perfect solution for you. It's open-source, it uses React, Node, GraphQL and TypeScript. It deploys your apps to your AWS cloud and runs on top of Lambda functions.

Webiny sponsor

[Next.js 9.5 Released](https://nextjs.org/blog/next-9-5) — [Vercel](https://vercel.com) CEO [Guillermo Rauch](https://twitter.com/rauchg) is excited about the new release: incremental static regeneration, custom base paths, rewrites and redirects, [webpack 5](https://webpack.js.org) support, and [much more](https://twitter.com/rauchg/status/1287792732472135680).

Next.js

[tinykeys: A Tiny, Modern Library for Keybindings](https://jamiebuilds.github.io/tinykeys/) — Has a reallly simple API and the page includes live use of some specific bindings that you can try out.

Jamie Kyle
