[Decorators for ES6 Classes Proposal (Mostly) Reaches Stage 3 at TC39](https://github.com/tc39/proposal-decorators) — We first linked to this proposal three years ago but it’s now [conditionally made it to stage 3](https://github.com/tc39/proposal-decorators/pull/454) (with some tweaks required) and people are [very excited](https://twitter.com/wycats/status/1508535701352038402). [React 18 Released](https://reactjs.org/blog/2022/03/29/react-v18.html) — [The plan](https://reactjs.org/blog/2021/06/08/the-plan-for-react-18.html) was for v18 to be an easy upgrade even if there were many changes under the hood.. and despite the introduction of a new concurrent renderer, Suspense, new hooks, automatic batching, and more, they’ve done a great job keeping the ship steady. [The React 18 upgrade guide](https://reactjs.org/blog/2022/03/08/react-18-upgrade-guide.html) covers some things you need to consider, and if you’re a completist, [the changelog](https://github.com/facebook/react/blob/main/CHANGELOG.md#1800-march-29-2022) digs deep.

## 📥 Reader Submissions

In recent months, we've been receiving an increasing amount of submissions from readers (thanks!) and while we use many of them, we don't have enough time to review *everything* to the right level. Given the backlog, we want to try out a new section to share things from our inbox to you more quickly:

- [Golgi](https://github.com/robtweed/golgi) by Rob Tweed is an interesting take on a UI framework built around [Web Components](https://developer.mozilla.org/en-US/docs/Web/Web_Components) and with no build steps required.
- Maxim Orlov has written about [how to run concurrent tasks within a defined limit](https://maximorlov.com/parallel-tasks-with-pure-javascript/) using purely JavaScript.
- [spotify-mini](https://github.com/rocktimsaikia/spotify-mini) is a Node wrapper around Spotify's Web API.
- [neo.mjs](https://github.com/neomjs/neo) (which has just hit [version 4.0](https://github.com/neomjs/neo/releases/tag/4.0.0)) is a performance-oriented framework built around workers and its team is continuing to [implement cool looking stuff.](https://itnext.io/predictive-offline-support-for-assets-you-have-not-used-yet-aeeccccd3754)

If *you* have something to submit, email `jsw@peterc.org` or just hit reply to any issue of this newsletter and we'll see it :-)
