[Anatomy of the TanStack npm Compromise](https://javascriptweekly.com/link/185001/web) — A new strain of the [Shai-Hulud worm](https://javascriptweekly.com/link/185002/web) pushed malicious versions of TanStack packages to npm yesterday (containing a tripwire that would delete files if it detected token revocation), though [it hit ~170 other packages too](https://javascriptweekly.com/link/185003/web). **Maintainer credentials weren’t stolen**, with the attack instead chaining `pull_request_target` abuse, cache poisoning, and OIDC token theft from CI *memory*.

Tanner Linsley

❓ **What should you do?** Consider an install-time cooldown (e.g. with `npm config set min-release-age=7` or [pnpm](https://javascriptweekly.com/link/185004/web)'s `minimumReleaseAge`), as the packages were only compromised for 26 minutes. Plus, audit your GitHub Actions workflows for security issues with a tool like [zizmor](https://javascriptweekly.com/link/185005/web).
