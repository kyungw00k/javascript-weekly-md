# JavaScript Weekly Issue 640: May 25, 2023

## Top Stories

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/iqzrtl8fc4k7gejg6e4f.jpg)

[DeviceScript: TypeScript for Tiny Thingamabobs](https://microsoft.github.io/devicescript/) — DeviceScript is a new Microsoft effort to take the TypeScript experience to low-resource microcontroller-based devices. It’s compiled to a custom VM bytecode which can run in such constrained environments. (A bit like Go’s [TinyGo.](https://tinygo.org/)) It’s aimed at VS Code users but there’s a [CLI option](https://microsoft.github.io/devicescript/getting-started/cli) too.

Microsoft

[The State of Node.js Performance in 2023](https://blog.rafaelgss.dev/state-of-nodejs-performance-2023) — Node 20 gets put through its paces against 18.16 and 16.20 with a few different benchmark suites running on an EC2 instance. It goes into a *lot* of depth that’s worth checking out, but if you haven’t got time, the conclusion is *“Node 20 is faster.”* Good.

Rafael Gonzaga

![](https://copm.s3.amazonaws.com/162049cf.png)

[Lightning Fast JavaScript Data Grid Widget](https://bryntum.com/products/grid/?utm_source=newsletter&utm_medium=email&utm_id=jsweekly-primary-20230519-grid) — Try a professional JS data grid component which lets you edit, sort, group and filter datasets with fantastic UX & performance. Includes a TreeGrid, API docs and lots of demos. Seamlessly integrates with React, Angular & Vue apps.

Bryntum Grid sponsor

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/akhsklce7qehwydcmm15.jpg)

[Deno 1.34: Now `deno compile` Supports npm Packages](https://deno.com/blog/v1.34) — [Deno](https://deno.com/runtime) isn’t Node, but it increasingly likes to wear a Node-shaped costume. This release focuses on npm and Node compatibility and Deno’s `compile` command (for turning projects into single binary executables) now supports npm packages too which opens up a lot of use cases.

The Deno Team
