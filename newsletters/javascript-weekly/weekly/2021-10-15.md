## Top Stories

[Parcel v2 Released](https://parceljs.org/blog/v2/) — Parcel, a fast zero-config webapp build tool, first came out in 2017 ([issue 364](https://javascriptweekly.com/issues/364)!) and took the JavaScript world by storm. v2 is a full rewrite that takes the same principles from v1 but extends them to fit projects of ‘any size and complexity’ by way of tree shaking by default, an all new plugin-based system, bundling via ES modules, automatic code splitting, image asset optimization, and, naturally, more.

Parcel Team

[The *Remix* Project Gets $3M in Seed Funding](https://remix.run/blog/seed-funding-for-remix) — The folks behind *React Training* and *React Router* have been working on a full-stack JavaScript framework called *Remix* which, till now, has only been available to paying supporters (who speak [very highly of it](https://dev.to/dabit3/a-first-look-at-remix-run-449a)). This funding round opens up a lot of new possibilities including the imminent open sourcing of the framework – good news for all of us.

**QUICK BITS:**

- The latest branch of the V8 JavaScript Engine, [V8 9.6](https://v8.dev/blog/v8-release-96), has been created. Very little headline news for this one, with support for using external references from JavaScript within WebAssembly modules being the only noted feature.
- [Node 16.11.1, 14.18.1, and 12.22.7](https://nodejs.org/en/blog/vulnerability/oct-2021-security-releases/) have been released to fix two HTTP request smuggling vulnerabilities.
- On Twitter, people shared [what things they've stopped doing in JavaScript](https://twitter.com/jamesqquick/status/1448060114053763076) as they've become more experienced.
- Serverless, Inc. has opened up the [public preview of Serverless Cloud](https://www.serverless.com/blog/introducing-serverless-cloud-public-preview), a serverless platform aimed specifically at JavaScript developers (for now).

[ESLint v8.0.0 Released](https://eslint.org/blog/2021/10/eslint-v8.0.0-released) — ESLint is a popular pluggable and configurable linter tool for maintaining JavaScript code quality. v8.0 brings it up to ES2022 standards. If you’re upgrading from an older version, a [migration guide](https://eslint.org/docs/user-guide/migrating-to-8.0.0) is available.
