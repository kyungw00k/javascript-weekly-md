# JavaScript Weekly Issue 661: November 2, 2023

## Top Stories

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/themuolqxrxycirsybdz.jpg)

[Next.js 14 Released](https://nextjs.org/blog/next-14) — Unveiled at last week’s *Next.js Conf*, v14 caused a *lot* of community discussion (not least [on Hacker News](https://news.ycombinator.com/item?id=38028480)), largely surrounding *Server Actions* being declared stable and the 'backend-meets-frontend' opportunities this opens up. A new *partial prerendering* feature is also in preview, allowing for dynamic responses but with the speed of an initial static response.

Lee Robinson and Tim Neutkens

💡 *The New Stack* has [an overview of the release](https://thenewstack.io/vercels-next-js-14-introduces-partial-pre-rendering/), and *Focus Reactive* has [a (highly visual) recap of *Next.js Conf* itself](https://focusreactive.com/recap-of-next-js-14-announcement-and-2023-conference-highlights/) – recommended if you didn't watch the stream.

Speakeasy sponsor

[Remix ❤️ Vite: Remix 2.2 Introduces Vite Support](https://remix.run/blog/remix-heart-vite) — [Remix](https://remix.run/) is a popular full-stack JavaScript framework that began as a paid product, but has been open source for two years now. If you found [Remix’s compilation](https://remix.run/docs/en/main/discussion/introduction) approach opaque before, we have great news: *“With Vite, Remix is no longer a compiler. Remix itself is just a Vite plugin.”* This post tells the full story.

Cattori and Dalgleish (Remix)

- 🗳 The folks behind the *State of JavaScript* survey have unveiled a new [*State of React* survey](https://survey.devographics.com/en-US/survey/state-of-react/2023) to take. As with their other surveys, you get to test your knowledge as well as provide useful insights.
- 🔁 [Shadow](https://goose.icu/introducing-shadow/) is a new, experimental browser engine built in JavaScript itself. It renders its output in your usual browser using an HTML canvas. [Source code.](https://github.com/canadahonk/shadow)
- 🔐 GitHub is [scanning all public npm packages](https://github.blog/changelog/2023-10-26-secret-scanning-scans-public-npm-packages/) for leaked secrets. Note that if secrets are found, the *provider* associated with those secrets is notified.
- ❄️ [WinterJS](https://wasmer.io/posts/announcing-winterjs-service-workers) is a new JS Service Workers server, powered by Rust & SpiderMonkey.
- 🐥 Uh-oh, it's [*Flappy Bird* implemented in TypeScript types.](https://zackoverflow.dev/writing/flappy-bird-in-type-level-typescript)

**RELEASES:**

- [Astro 3.4](https://astro.build/blog/astro-340/) – The Web framework gains page partials, a new experimental dev overlay, and more.

- [Rollup 4.2](https://github.com/rollup/rollup), [Cypress 13.4](https://docs.cypress.io/guides/references/changelog#13-4-0), [Ember.js 5.4](https://github.com/emberjs/ember.js), [Stencil 4.7](https://github.com/ionic-team/stencil/releases/tag/v4.7.0)

## 💛 And something else we love

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/wq1kinztbo42a8lgbgwz.jpg)

[Val Town 3.0: A Social Way to Write and Deploy TypeScript](https://www.val.town/) — *“If GitHub Gists Could Run, and AWS Lambda Were Fun”* is a fantastic description for this online platform where you write tiny bite size functions (‘val’s) to be run in V8 isolates. You can connect them together, schedule them, serve them up over HTTP, and more. [Version 3](https://blog.val.town/blog/introducing-val-town-v3) is a big jump for the platform, adding JSX support, more standardized JavaScript approaches to solving problems, and the ability to edit and run your ‘vals’ *locally.*

Steve Krouse
