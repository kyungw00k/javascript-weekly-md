# JavaScript Weekly Issue 527: February 26, 2021

## Top Stories

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60/ul7dodq1geq6u1q3hd1x.jpg)

[JavaScript Performance Beyond Bundle Size](https://nolanlawson.com/2021/02/23/javascript-performance-beyond-bundle-size/) — Nolan ponders why ‘bundle size’ is such a common thing to focus on when it comes to JavaScript performance and shares a collection of useful tools to study it further, if you wish, before moving on to more significant concerns ‘beyond the bundle.’ A really good read. Alex Russell [reminds us](https://twitter.com/slightlylate/status/1364362292670857220) to test on real mobile hardware too.

Nolan Lawson

[The Definitive Guide to Streams](https://web.dev/streams/) — One of those posts that will bring you up to speed on a not often discussed area of JavaScript but which is really useful to understand! The Streams API allows you to programmatically read, write, and transform streams of data in a flexible way.

Thomas Steiner

![](https://copm.s3.amazonaws.com/ae169a71.jpg)

[A Friendly Introduction to Functional Programming with JavaScript by Anjana Vakil](https://frontendmasters.com/courses/functional-first-steps/?utm_source=javascriptweekly&utm_medium=email&utm_campaign=functionalprogramming) — Learn what Functional Programming (FP) is and how it differs from other programming styles (such as OO and imperative), and use key concepts of functional to improve your everyday coding.

Frontend Masters sponsor

[TypeScript 4.2 Released](https://devblogs.microsoft.com/typescript/announcing-typescript-4-2/) — The extension (a superset, if you will) of JavaScript that adds static types and type-checking takes another step forward. 4.2 improves how type aliases are handled, adds leading/middle rest elements to tuple types, and more. The release notes are super thorough, as always.

Daniel Rosenwasser (Microsoft)

[10 Years of Open-Source Visualization with D3.js](https://observablehq.com/@mbostock/10-years-of-open-source-visualization) — The creator of [D3.js](https://d3js.org/) (the popular data visualization library) reflects on ten years since D3 1.0 was released with what he’s learnt from both developing D3 and leading a prominent project for so long. Congratulations, Mike!

Mike Bostock
