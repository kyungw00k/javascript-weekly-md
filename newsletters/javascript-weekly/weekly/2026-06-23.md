# Babel 8.0, Vite 8.1, and TypeScript 7.0 RC

> JavaScript Weekly — Issue #791

## Top Stories

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/nmuo98ex7rzsd4bj7hcu.jpg)

[Desktop Apps With `deno desktop`](https://docs.deno.com/runtime/desktop/) — Deno 2.9 (or the 'canary' build *now*) can turn JavaScript projects into self-contained apps on macOS, Windows, and Linux. Unlike Electron, you can opt to use the default OS WebView *or* a bundled Chromium backend, plus you get cross-compilation and automatic support for apps built on frameworks like Next.js and SvelteKit.

The Deno Project

📉 It's not in the post, but a `--compress` option gets packaged app sizes down a lot – from 65MB to 19MB in my test with a basic app.

[Catch Runtime Bugs Before Your JS PR Merges](https://www.greptile.com?utm_source=javascriptweekly&utm_medium=paid-community&utm_campaign=javascriptweekly_primary_jun23) — Some JavaScript bugs only show up after the app runs. Greptile sponsor

[Babel 8.0 Released: The JavaScript Transpiler](https://babeljs.io/blog/2026/06/16/8.0.0/) — It’s been *eight years* since [Babel 7.0](https://babeljs.io/blog/2018/08/27/7.0.0/), and despite the rise of tools like SWC and Oxc, weekly downloads of the `@babel` packages have increased 380x since. v8.0 is a modernization release that goes ESM-only and targets ~ES2023 by default (vs ES5 before).

Babel Team

[TypeScript 7.0 Release Candidate](https://devblogs.microsoft.com/typescript/announcing-typescript-7-0-rc/) — A step closer for the Go-powered TypeScript compiler that promises ~10x faster build performance. *(I've seen people on social media getting very excited by this.)*

Daniel Rosenwasser (Microsoft)

**IN BRIEF:**

- Cloudflare has added the ability to [create and deploy Workers on 'temporary accounts'](https://blog.cloudflare.com/temporary-accounts/). Ostensibly 'for AI agents', but you can use them yourself too, and get a Worker running for 60 minutes with no strings attached.
- TC39 delegate Luca Casonato has made [a proposal to add built-in localized time formatting](https://github.com/whatwg/html/issues/12591) to `<time>`, akin to GitHub's [`<relative-time>` element](https://github.com/github/relative-time-element).
- 🤖 [MDN has introduced an official MCP server](https://developer.mozilla.org/en-US/blog/introducing-mdn-mcp-server/) to provide better access to MDN's up-to-date JS and frontend docs to coding agents.
- The Bun team has proposed [adding shared-memory threads to JavaScriptCore](https://github.com/oven-sh/WebKit/pull/249) (a more traditional threading approach versus [worker threads](https://nodejs.org/api/worker_threads.html#worker-threads)).

**RELEASES:**

- ⭐ [Vite 8.1](https://vite.dev/blog/announcing-vite8-1) – Adds experimental support for 'bundled dev mode' which radically speeds up dev server startup and full reloads on large apps. Also adds [WASM/ESM integration](https://github.com/WebAssembly/esm-integration/blob/main/proposals/esm-integration/README.md) support.
- ⭐ [Astro 7.0](https://astro.build/blog/astro-7/) – The popular web framework gets faster build times, with `.astro` compilation and Markdown/MDX processing going through a Rust-powered pipeline. *Advanced Routing* also gives you full control over the request pipeline.
- [pnpm 11.7](https://pnpm.io/blog/releases/11.7) and [11.8](https://github.com/pnpm/pnpm/releases/tag/v11.8.0) – `pnpm install` gets a `--dry-run` option, `--frozen-store` adds support for read-only package stores, and more.
- [Node.js v26.3.1 (Current)](https://nodejs.org/en/blog/release/v26.3.1), [v24.17.0 (LTS)](https://nodejs.org/en/blog/release/v24.17.0), and [v22.23.0 (LTS)](https://nodejs.org/en/blog/release/v22.23.0)

## 📖 Articles and Videos

[How an Underrated Refactor Saved 90% Memory Usage](https://tanstack.com/blog/tanstack-table-v9-memory-performance) — The tale of how [TanStack Table](https://tanstack.com/table/latest) v9 uses less memory than v8 on large tables, thanks to a simple idea: storing methods on shared prototypes instead of creating them for every object. Benchmarks included, plus a pattern other libraries can copy.

Kevin Van Cott (TanStack)

[You Already Know Enough JavaScript to Ship a Real Mobile App](https://try.expo.dev/jsweekly) — Build for iOS and Android from one codebase with React. Expo handles builds, the App Store, and updates.
