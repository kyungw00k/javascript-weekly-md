**IN BRIEF:**

- **DEBUGGING TIP:** David Walsh shows off Chrome's [rather useful `monitorEvents` and `monitor` helper functions](https://davidwalsh.name/monitorevents) for being notified when certain events occur or if a certain function is ever called. I didn't know about these.. and even Ben Nadel in the comments said: *"Oh my chickens, this is amazing"* 😆
- Another month, [another VS Code release.](https://code.visualstudio.com/updates/v1_71) Context-maintaining 'sticky scroll' is no longer an experimental feature. Ships with TypeScript 4.8. Oh, and rounded buttons 😏
- 18 months after jQuery 3.6.0 was released, [jQuery 3.6.1 is here](https://blog.jquery.com/2022/08/26/jquery-3-6-1-maintenance-release/) as a maintenance release. The fixes and improvements are minor indeed, but I can't help but feel comforted by its continued existence 🥰

## 📒 Articles & Tutorials

[Are Default Exports in JavaScript Modules Terrible?](https://www.lloydatkinson.net/posts/2022/default-exports-in-javascript-modules-are-terrible/) — Lloyd thinks so and points out that default exports can lead to mismatched and confusing names, so prefers named exports instead. As always, though, it comes down to how you use the feature.

Lloyd Atkinson

[Implementing a Promisable `setTimeout`](https://yieldcode.blog/post/implementing-promisable-set-timeout) — One of those explorations that’s more about learning about how things work under the hood, rather than to do anything useful. Dmitry Kudryavtsev

## 🛠 Code & Tools

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60/zpdawxjw79qr580b89cp.jpg)

[Lusift: Library to Create Product Walkthroughs for Your Web App](https://lusift.vercel.app/) — Happy to go vanilla, but has React and Vue integrations too. [GitHub repo](https://github.com/lusift/lusift). Other options in this space are [Shepherd](https://shepherdjs.dev/) and [React Joyride](https://react-joyride.com/).

[Derive Type: Generate Type Definitions for JS Automatically From Tests](https://github.com/David-Kunz/derive-type) — Derive types dynamically by running tests to capture value combinations – the idea is to help you manage things while developing functions, particularly if you’re *not* able to use TypeScript ([more from the author on this](https://www.reddit.com/r/javascript/comments/wzx52i/derivetype_automatically_generate_typescript/im56pcj/)). There’s [▶️ a screencast](https://www.youtube.com/watch?v=gdz_X0b5SnM) that explains more.

[Hyper Fetch: A Turbocharged Fetch Library](https://hyperfetch.bettertyped.com/) — The creator bills it as *“a mix of Axios and react-query in one, with advanced persistence options.”* It’s backend agnostic and offers queues, caching, persistence, and even offline support out of the box. Maciej Pyrc et al.

[Partytown 0.7: Run Third Party Scripts in a Web Worker](https://partytown.builder.io/) — The idea is simple: run resource intensive scripts on a separate thread (in a web worker) to let the main thread remain responsive. v0.7 adds an option to [execute *certain* scripts on the main thread](https://github.com/BuilderIO/partytown/pull/235) which can be useful in some situations (an example is provided). [GitHub repo](https://github.com/BuilderIO/partytown).
