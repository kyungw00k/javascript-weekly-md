# JavaScript Weekly Issue 658: October 12, 2023

## Top Stories

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/rhix34vjuwb86xzut5k2.jpg)

[Speeding Up the JS Ecosystem: The Barrel File Debacle](https://marvinh.dev/blog/speeding-up-javascript-ecosystem-part-7/) — Marvin continues his tour through the world of JavaScript performance fixes with a look at how some innocent looking code can make tools run slower than they should. Test runners and many import cycle detection tools are most affected.

Marvin Hagemeister

🌊 [Building a Water/Fluid Simulation in JavaScript](https://kyndinfo.notion.site/Fluid-Simulation-f0516d9d12e245a08ae5c7545ac822dd) — Hard to explain, but a lot of thought and care has gone into this tutorial of sorts. The live demos are nice, too. Uses [p5.js](https://p5js.org/) behind the scenes.

Kenichi Yoneda

![](https://copm.s3.amazonaws.com/a1148dc7.png)

[Strongly Typed and in the ⛅: Meet EdgeDB 4.0](https://www.edgedb.com/launch?lang=ts) — EdgeDB is an open-source database focused on developer experience. It features a high-level, modern data model, integrated schema migrations, and a TypeScript query builder that makes it easy to write advanced, fast queries, putting ORMs to shame.

EdgeDB sponsor

[Angular and Qwik's Creator on How JS Frameworks Handle Reactivity](https://thenewstack.io/angular-qwik-creator-on-how-js-frameworks-handle-reactivity/) — A summary of Miško Hevery’s keynote from the *International JavaScript Conference* looking at reactivity across the major frameworks.

Loraine Lawson (The New Stack)

[A Web Server ‘Hello World’ Benchmark: Go vs Node vs Nim vs Bun](https://lemire.me/blog/2023/10/07/web-server-hello-world-benchmark-go-vs-node-js-vs-nim-vs-bun/) — The standard disclaimer applies: benchmarks are difficult and don’t always measure what you *should* care about.

Daniel Lemire

- 👀 Simon Willison has put together [a fantastic live demo](https://observablehq.com/@simonw/detect-objects-in-images) (in notebook format) showing off how to do client-side object detection in images using solely JavaScript and a neural network. *(Note: Be aware this uses a lot of CPU and may slow your browser down.)*
- ▶️ *ViteConf 2023* took place last week and [some of the talks](https://www.youtube.com/playlist?list=PLqGQbXn_GDmkOsHI7-Wrbv1GgAA4tJZhg) are already available on YouTube. [🐦 One key bit of news is about Rolldown](https://twitter.com/youyuxi/status/1709943106215530867), a Rust port of Rollup, meaning Vite will get even faster in the medium term.
- ⚙️ Stephen Röttger of the V8 team has blogged about efforts to [enable control-flow integrity (CFI) in V8](https://v8.dev/blog/control-flow-integrity), essentially a way to reduce the ability of nefarious third parties to exploit V8's control flow in order to trigger arbitrary shellcode.
- ⚠️ A look at [the potential dangers of regular expressions in JavaScript.](https://www.sonarsource.com/blog/vulnerable-regular-expressions-javascript/)
- 👾 [An interesting post mortem of a game](https://remvst.medium.com/path-to-glory-post-mortem-js13k-2023-be74a5272621) built in just 13KB for the JS13K contest.

**🎉 RELEASES:**

- [Electron 27](https://www.electronjs.org/blog/electron-27-0) – The cross-platform desktop app toolkit gains Chromium 118, Node.js 18.17.1, and V8 11.8, but loses macOS 10.13/14 support.
- [Fresh 1.5](https://deno.com/blog/fresh-1.5) – A Deno-native framework for building full-stack webapps. v1.5 adds support for *partials* – the ability to update a *portion* of an existing page.
- [Parcel 2.10](https://github.com/parcel-bundler/parcel/releases/tag/v2.10.0) – The zero-config build tool gets some *serious* performance improvements (7x faster on large projects). Devon Govett wrote [🐦 a Twitter thread explaining how.](https://twitter.com/devongovett/status/1712169214872867288)
- [Bun 1.0.5](https://bun.sh/blog/bun-v1.0.5) – Mostly bug fixes and gentle improvements for the performance-oriented JS runtime. No HTTP/2 yet, but it's just around the corner..
- [Solid 1.8](https://github.com/solidjs/solid/releases/tag/v1.8.0) – Declarative and performant reactivity for building UIs.
