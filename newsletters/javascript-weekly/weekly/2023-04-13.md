## Top Stories

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/fbcblmd7d5qxno7hqc9o.jpg)

[The JavaScript Equality Table Game](https://eqeq.js.org/) — *Minesweeper* will feel like a walk in the park after this reminder of the horrors of JavaScript’s `==`. If you need to go in depth, [Section 7.2.14](https://262.ecma-international.org/9.0/#sec-abstract-equality-comparison) of the ECMAScript spec will help, but otherwise? Stick to three equals (`===`) unless you have a good reason not to.

Reinis Ivanovs

[htmx 1.9 Released](https://htmx.org/posts/2023-04-11-htmx-1-9-0-is-released/) — htmx ([homepage](https://htmx.org/)) is an increasingly popular library *outside* of the JavaScript space as it lets folks use things like WebSockets, SSE, AJAX, and CSS transitions by marking up HTML rather than writing lots of JavaScript. v1.9 adds support for view transitions and generalized inline event handling. [The code examples](https://htmx.org/examples/) are worth a look – htmx makes a lot possible, with rather little tooling or markup needed.

htmx team

[The ECMAScript® 2023 Language Spec Steps Forward](https://tc39.es/ecma262/2023/) — After prematurely announcing the progression of the ES2023 spec in February, we can now announce: TC39 has approved the ECMAScript 2023 spec, and while it remains a candidate, it’s now a step closer to eventual ECMA General Assembly approval. The [finished proposals list](https://github.com/tc39/proposals/blob/main/finished-proposals.md) for 2023 now includes [Array find from last](https://github.com/tc39/proposal-array-find-from-last), [hashbang support](https://github.com/tc39/proposal-hashbang), [Symbols as WeakMap keys](https://github.com/tc39/proposal-symbols-as-weakmap-keys), and [change Array by copy](https://github.com/tc39/proposal-change-array-by-copy).
