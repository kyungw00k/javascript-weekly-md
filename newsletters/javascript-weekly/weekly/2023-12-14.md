[Oxlint: A New JS Linter '50-100x Faster' than ESLint?](https://oxc-project.github.io/blog/2023-12-12-announcing-oxlint.html) — Making tools as fast as possible is a common theme in the JavaScript world, and it's the MO for [the *JavaScript Oxidation Compiler toolkit.*](https://oxc-project.github.io/) *Oxlint* is one of their tools that’s now GA. It's still early days, but when Evan You [says](https://twitter.com/youyuxi/status/1734439543280128030) *“the performance is absolutely nuts”* when running against the Vue 3 codebase, we sit up and listen.

[Bun v1.0.18: The JS Runtime Preparing to Take Over the World](https://bun.sh/blog/bun-v1.0.18) — I *might* have overegged the headline, but Bun is on a serious mission to replace Node.js, at least, with [their 'one goal for 2024'](https://twitter.com/bunjavascript/status/1732945030007099510) being to *“flip the default backend JavaScript runtime from Node.js to Bun.”* They are also placing a [focus on shipping Windows support](https://twitter.com/bunjavascript/status/1730757658486157556) ASAP.

[Deno 1.39: The Return of WebGPU](https://deno.com/blog/v1.39) — Deno introduced WebGPU support [in Deno 1.8](https://deno.com/blog/v1.8#experimental-support-for-the-webgpu-api) but it was removed for various reasons. It’s *back* and gives you a tidy way to work with the GPU directly from JavaScript (behind the `--unstable-webgpu` flag, at least). Improvements to Node.js compatibility and various Deno APIs, as well as TypeScript 5.3, are included.

The Deno Team
