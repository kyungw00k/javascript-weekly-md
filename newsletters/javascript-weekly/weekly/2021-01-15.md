# JavaScript Weekly Issue 521: January 15, 2021

## Top Stories

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60/iwttkc0x71x8wbxabkps.jpg)

[The State of JS 2020 Survey Results](https://2020.stateofjs.com/en-US/) — 23,765 people from 137 countries took part in the recent *State of JS* survey and while there are some [common criticisms](https://dev.to/sachagreif/state-of-js-2020-common-criticisms-23id) of the project, the results are nonetheless interesting and we’ll be digging into some in forthcoming issues. Standouts include:

- Svelte took the [top frontend framework crown](https://2020.stateofjs.com/en-US/technologies/front-end-frameworks/) from React for developer satisfaction.
- [*Testing Library*](https://testing-library.com/) jumped straight to #1 for [testing libraries.](https://2020.stateofjs.com/en-US/technologies/testing/)
- [More developers than ever](https://2020.stateofjs.com/en-US/features/other-features/) are producing PWAs or using WebAssembly.
- 86% of respondents are [using *VS Code*](https://2020.stateofjs.com/en-US/other-tools/#text_editors) to work on their code.

State of JS

[Announcing TypeScript 4.2 Beta](https://devblogs.microsoft.com/typescript/announcing-typescript-4-2-beta/) — What’s new? Leading/middle rest elements in tuple types (e.g. `let bar: [boolean, ...string[], boolean]`), smarter type alias preservation, stricter checks for the `in` operator, and more.

Daniel Rosenwasser (Microsoft)

[Snowpack v3.0: Reimagining Web Development with ESM](https://www.snowpack.dev/posts/2021-01-13-snowpack-3-0) — You can now import any npm package on demand (with streaming imports), and there’s a new native JavaScript API. Oh, and it’s the one year anniversary of Snowpack’s 1.0 release! 🎉

Fred K. Schott

We asked Snowpack's creator, [Fred K Schott](https://twitter.com/FredKSchott), for a few words on [Snowpack 3.0](https://www.snowpack.dev/posts/2021-01-13-snowpack-3-0):

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60/lgvlusyoce36f5m6ygyj.jpg)

“I couldn't be more excited about this launch! Streaming imports are a game changer that simplify dependency management by fetching packages as soon as you need them: pre-built, pre-bundled and ready to run. We also shipped a first-class integration with `esbuild` to bundle Snowpack builds 100x faster than Webpack, Rollup or Parcel (seriously, look at [these benchmarks!](https://esbuild.github.io/)). If you haven't had a chance to check out Snowpack yet, v3.0 is a great chance to jump in.”

[Node Package Maintainers, Get Ready For ES Modules](https://blog.sindresorhus.com/get-ready-for-esm-aa53530b3f77) — ES modules are rapidly becoming a day to day reality for Node.js package maintainers, so spare a thought for Sindre who has about 20 million packages (*slight exaggeration*) to his name. Sindre shares some basics on why it’s important and what his plans are.

Sindre Sorhus
