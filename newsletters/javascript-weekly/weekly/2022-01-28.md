# JavaScript Weekly Issue 574: January 28, 2022

## Top Stories

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60/xbrtnhjvalt0al01p0nm.jpg)

[A Pipe Operator for JavaScript: Introduction and Use Cases](https://2ality.com/2022/01/pipe-operator.html) — In the State of JS 2020 survey, a *lot* of developers felt a pipe operator was missing from JavaScript. It’s been around a while, but there’s a [pipeline operator proposal](https://github.com/tc39/proposal-pipeline-operator) and it’s at stage two of the TC39 process. Dr. Axel explains why you’d want this, how it can work, and why JS *won’t* get F# style pipes.

Dr. Axel Rauschmayer

[On Porting the TypeScript Compiler to Go (vs Rust)](https://kdy1.dev/posts/2022/1/tsc-go) — The author is a Rust advocate (who created [swc](https://swc.rs/)) but recognizes that Go might be a better tool particularly for building the *type checker* and explains why here. This post also fuelled a rather [extensive Hacker News discussion](https://news.ycombinator.com/item?id=30074414) on the complications involved with many coming to Rust's defence.

DongYoon Kang

[Take the State of JavaScript 2022 Survey](https://stateofjs.com/) — Now several years in, the popular State of JavaScript survey returns for a joint 2021/22 run and, as ever, seeks to establish what libraries and frameworks developers are excited about. **IN BRIEF:**

- ⭐️ An engineer at Etsy [mentions Etsy migrated](https://twitter.com/sangster/status/1486382892326563845) all their React 15.6 code to [Preact](https://preactjs.com/) rather than going to React 16+. A [doc from 2020](https://github.com/mq2thez/blog/blob/main/upgrade-react-etsy/preact-vs-react.md) explained some of the internal thinking around this choice.
- The Deno team has [reflected on Deno's progress in 2021.](https://deno.com/blog/deno-in-2021) The [Angular team has done similarly,](https://blog.angular.io/angular-2021-recap-and-2022-preview-cb3067f76217) too.
- [TypeScript 4.6 is in beta](https://devblogs.microsoft.com/typescript/announcing-typescript-4-6-beta/) and focused on relatively minor technical enhancements, such as improved errors, a trace analyzer, and allowing code in constructors *prior* to calling `super()`.
- Ryan Florence of the Remix project notes on Twitter a *["palpable feeling that folks are ready for the post-React world"](https://twitter.com/ryanflorence/status/1486530142507655173)*.
- Mike Melanson asks [if 2022 will be a 'golden age' for full-stack JavaScript](https://thenewstack.io/will-2022-be-a-golden-age-for-full-stack-javascript/)?
