# JavaScript Weekly Issue 750: August 29, 2025

## Top Stories

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/h7599gxmqq4qxdqo226a.jpg)

[An Illustrated Guide to Big O and Time Complexity](https://samwho.dev/big-o/) — A fantastic JavaScript-oriented, interactive, visual essay about [Big O notation](https://en.wikipedia.org/wiki/Big_O_notation) and its role in describing algorithmic complexity. This is a beautiful bit of work, even if you’re already wise to `O(log n)` and `O(n^2)`.

Sam Rose

💡 Sam is making a name for himself with his interactive essays – check out the [load balancing](https://samwho.dev/load-balancing/), [memory allocation](https://samwho.dev/memory-allocation/), [hashing](https://samwho.dev/hashing/), and [bloom filter](https://samwho.dev/bloom-filters/) ones too.

[JavaScript’s Trademark Problem](https://2ality.com/2025/08/javascript-trademark.html) — Dr. Axel discusses Oracle's hold over the ‘JavaScript’ trademark, including the history and present-day [legal attempts to challenge Oracle.](http://javascript.tm/) Fireship has also just published [▶️ a video about the battle](https://www.youtube.com/watch?v=pOF11EDprxc) and there’s now [a GoFundMe to raise funds](https://www.gofundme.com/f/help-us-challenge-oracles-javascript-trademark) to continue the fight.

Dr. Axel Rauschmayer et al.

[Tests Are Dead. Meticulous AI Is Here](https://www.meticulous.ai/?utm_source=javascript_weekly_primary&utm_campaign=august29th2025) — Meticulous creates and maintains a continuously evolving E2E UI test suite with zero developer effort. Built on Chromium with a deterministic engine, it’s the only testing tool that eliminates flakes. Relied on by Dropbox, Notion, and Wiz.

Meticulous AI sponsor

[New in ESLint v9.34.0: Multithread Linting](https://eslint.org/blog/2025/08/multithread-linting/) — A common feature of tools like [Rslint](https://rslint.rs/), [Biome](https://biomejs.dev/), and [Oxlint](https://oxc.rs/blog/2025-08-17-oxlint-type-aware.html), is being '*N*x times faster' than ESLint, but the [latest ESLint](https://eslint.org/blog/2025/08/eslint-v9.34.0-released/) has a trick up its sleeve and can now process multiple files at the same time, dramatically reducing lint times for large projects.

Francesco Trotta

[Say Bye with JavaScript Beacon](https://hemath.dev/blog/say-bye-with-javascript-beacon/) — The [Beacon API](https://developer.mozilla.org/en-US/docs/Web/API/Beacon_API) has been supported in all mainstream browsers for years now and provides a way to send a guaranteed (even if a page is being unloaded) non-blocking request to a web server when no response is needed.
