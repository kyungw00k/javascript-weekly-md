# JavaScript Weekly Issue 586: April 22, 2022

## Top Stories

Lots of bits and pieces this week like running JavaScript on MS-DOS(!?), a deep dive on a V8 optimization, writing React apps on top of Ruby on Rails, a JS runtime for a tiny microcontroller, and, oh, a huge Node release..

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60/qwfonw0ametgqbvq807q.jpg)

[Lexical: An Extensible Text Editor Library (That Does Things Differently)](https://lexical.dev/) — Fresh from Meta (or Facebook, as my brain still calls it) comes a new text editor framework with accessibility, performance, and reliability at its heart. 22KB gzipped, supports React 18 (but can be used in a vanilla fashion), and a (native) iOS version is to follow. Remind you of [Draft.js](https://draftjs.org/)? Think of this as the next generation: Meta is – they’re replacing Draft with Lexical in their apps.

Meta / Facebook

[Node.js 18 (Current) Released](https://nodejs.org/en/blog/announcements/v18-release-announce/) — We get V8 10.1, Undici-powered global Fetch API support, Web Streams API support, Blob and BroadcastChannel APIs, the new test runner module, and the ability to create your own Node.js binaries with custom V8 startup snapshots. Node 18 goes LTS from October, but gets all the latest features until then.

The Node.js Team
