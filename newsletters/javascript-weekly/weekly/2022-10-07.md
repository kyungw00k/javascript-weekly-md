[Axios 1: The Long Standing HTTP Client Library](https://github.com/axios/axios/releases/tag/v1.0.0) — After eight years, 100,000 stars, and a presence in thousands of projects, Axios remains popular, yet v1.0 has only just landed (technically [v1.1.2](https://github.com/axios/axios/releases/tag/v1.1.2) now – after some fixes and a [a rather problematic change](https://github.com/axios/axios/issues/5038)). The [Fetch API](https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API) has eaten into Axios' market in recent years but, a bit like jQuery, Axios still wraps up a lot of functionality into an API many developers like. ([Project homepage](https://axios-http.com/).)

Axios Project

🤔 If you wonder why people use Axios when the Fetch API is widely supporters, [interceptors](https://axios-http.com/docs/interceptors) are just one of many features its fans cite.

['I Turned JS Into a Compiled Language (for Fun and Wasm)'](https://surma.dev/things/compile-js/) — You can never be bored observing Surma’s work which continues to provoke ideas and discussion ([such as with Web Workers](https://surma.dev/things/when-workers/)). Now he’s trying to run JavaScript *in* WebAssembly, which required turning JavaScript into C++ first. Eek! Good idea? Does it matter? Spoiler: *“I think this technique is a dead end”* but it's a neat journey anyway and he explains how [AssemblyScript](https://www.assemblyscript.org/) suggests a more promising way forward.

Surma

🎂 [Ten Years of TypeScript](https://devblogs.microsoft.com/typescript/ten-years-of-typescript/) — Even if you don’t use it, TypeScript's influence on the JS landscape is significant, and it’s now *ten years old.* (It was our top feature in issue 99 along with the release of jQuery Mobile 1.2.0! 😏). TypeScript's PM reflects on how its ideas have stood the test of time and looks forward to another ten years.

Daniel Rosenwasser (Microsoft)

🕹 [The js13kGames 2022 Winners](https://github.blog/2022-10-06-js13k-2022-winners/) — The eleventh annual js13kGames coding competition that challenges participants to create games in 13KB or less of JavaScript has wrapped up and [here are the winners](https://js13kgames.com/#winners). If you have the time, there’s a lot to learn from the shared code, or you could just [play the winning game](https://js13kgames.com/entries/dante) – a surprisingly slick 3D puzzler platformer.

[The Future of the Web is on the Edge (Says *Deno*)](https://deno.com/blog/the-future-of-web-is-on-the-edge) — [Deno](https://deno.land/) is not merely an alternative server-side runtime for JavaScript – it's at the heart of *Deno Deploy*, a serverless platform for taking JavaScript 'to the edge.'
