# JavaScript Weekly Issue 552: August 20, 2021

## Top Stories

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60/wncpadxdoj0bgw7wql8r.jpg)

[Ruby on Rails' Creator on Modern Web Apps without JS Bundling or Transpiling](https://world.hey.com/dhh/modern-web-apps-without-javascript-bundling-or-transpiling-a20f2755) — David Heinemeier Hansson (the original creator of the popular [Ruby on Rails](https://rubyonrails.org/) framework) has explained where he sees the future of JavaScript in Rails-based apps and it involves using import maps and *not* needing bundlers like Webpacker – instead apps will just pull in the ES modules needed. He also shows off how it works in [this 27-minute screencast](https://www.youtube.com/watch?v=PtxZvFnL2i0).

David Heinemeier Hansson (DHH)

[An Absurd Future for SQL on the Web?](https://jlongster.com/future-sql-web) — [absurd-sql](https://github.com/jlongster/absurd-sql) is a project aiming to provide a persistent backend for SQLite on the web. It uses IndexedDB but with sql.js in front so you can use SQLite in a natural but persisted way from webapps. Some interesting thoughts here.

James Long
