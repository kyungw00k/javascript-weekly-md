# What's new in ECMAScript 2026 (and usable today)

> JavaScript Weekly — Issue #793

## Top Stories

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/uspa69xtit607rmfzdnq.jpg)

[Vite+ Beta: A Web Dev Toolchain Behind One Command](https://voidzero.dev/posts/announcing-vite-plus-beta) — Vite+ is the [Vite](https://vite.dev/) team’s ‘unified toolchain’ that brings Vite, Vitest, Oxlint, and similar tools together under a single `vp` command, whether for running a dev server, tests, formatting, or bundling.

VoidZero

💡 Vite+ was originally intended to be a commercial project to fund work on Vite and related projects, but was open sourced under the MIT license earlier this year.

![](https://res.cloudinary.com/cpress/image/upload/c_limit,w_480,h_480,q_auto/copm/eb4e5347.png)

[Still Writing Tests Manually? Meticulous AI Is Here](https://www.meticulous.ai/?utm_source=jsweekly&utm_medium=newsletter&utm_campaign=26q3&utm_content=primary) — Notion, Dropbox, Wiz and LaunchDarkly now use a testing paradigm they can’t work without. Built by former Palantir engineers, Meticulous automatically creates an evolving suite of E2E UI tests, delivering exhaustive coverage with no developer effort.

Meticulous sponsor

[What's New in ECMAScript 2026](https://pawelgrzybek.com/whats-new-in-ecmascript-2026/) — The 876 page [ECMAScript 2026 spec](https://ecma-international.org/publications-and-standards/standards/ecma-262/) was approved last week, and this round-up covers several main new features that made the cut, from `Array.fromAsync` to native `Uint8Array` Base64/Hex conversion, with examples for each. All are already in browsers and runtimes (except `Math.sumPrecise` in Node) so you can likely use them today.

Pawel Grzybek

**IN BRIEF:**

- 🕹️ [js1024, the annual 1KB code-golf competition, is open for entries](https://js1024.fun/#2026). The theme is 'dreaming', and you've got 1,024 bytes to make something neat. Here's [last year's winner](https://js1024.fun/demos/2025/11/bar) to get a feel for the caliber.
- There's [a TC39 proposal to standardize a `code` property on `Error`](https://github.com/tc39/proposal-error-code-property) (long a de-facto Node convention). Its champion, James Snell, hopes to advance it at this month's plenary.

- [webpack-dev-server 6.0](https://github.com/webpack/webpack-dev-server/releases/tag/v6.0.0) – The official webpack dev server with live reloading upgrades to Express 5 and native ESM.
- [Dependency Cruiser 18.0](https://github.com/sverweij/dependency-cruiser) – Tool to validate and graph project dependencies.
- [npm 11.18.0](https://github.com/npm/cli/releases/tag/v11.18.0), [pnpm 11.10](https://pnpm.io/blog/releases/11.10)

## 📢 Elsewhere in the ecosystem

- 🤖 [Safari has introduced an MCP server](https://webkit.org/blog/18136/introducing-the-safari-mcp-server-for-web-developers/) in its latest preview release, providing agents with a way to connect to a Safari window and emulate the user experience while debugging.
- [`shadcn/ui` has switched to Base UI as its default component library](https://ui.shadcn.com/docs/changelog) (from Radix). Radix isn't going away, but [Base UI](https://base-ui.com/) is recommended for new projects.
- [PocketJS](https://pocketjs.dev/) is a project that brings a JavaScript VM, Vue Vapor, and Solid UI into an 8MB memory budget, and can, curiously, run on the Sony PSP, as well as in WASM and natively on macOS.
- 📈 [Commit History](https://commit-history.com/) is an online tool that creates charts of a user's GitHub commits over time. A nice addition for your [profile README](https://docs.github.com/en/account-and-profile/how-tos/profile-customization/managing-your-profile-readme), or check out [Sindre Sorhus's chart](https://commit-history.com/sindresorhus) to see the legendary maintainer's consistency over the years.
- 🎉 [PGlite](https://pglite.dev/), a WebAssembly build of Postgres for use in JavaScript environments, [celebrates reaching 10 million weekly downloads](https://electric.ax/blog/2026/06/25/pglite-reaches-10-million-weekly-downloads) while explaining the use cases and how to get started with it yourself.
