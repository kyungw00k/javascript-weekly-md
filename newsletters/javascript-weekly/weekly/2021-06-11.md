## Top Stories

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60/nk7ji6gs1qbeyekjuuys.jpg)

[Introducing Astro: Ship (Even) Less JavaScript](https://astro.build/blog/introducing-astro) — From the creator of [Snowpack](https://www.snowpack.dev/) and [Skypack](https://www.skypack.dev/) comes **a new 'bring your own framework' static site builder** where the site is rendered to static HTML and only loads JavaScript as-needed using an “islands architecture” to allow partial hydration. It's always fun to see these folks experimenting with how we approach problems. Keen to learn more? [This 50 minute talk + live demo](https://www.youtube.com/watch?v=mgkwZqVkrwo) helps *a lot.*

Fred K. Schott

[The Plan for React 18](https://reactjs.org/blog/2021/06/08/the-plan-for-react-18.html) — The React team has outlined some exciting things on the roadmap for React 18 (due in its final form in *'several months' time'*):

- Cassidy Williams has written [an accessible 'Now what?' introduction](https://dev.to/cassidoo/react-18-alpha-is-out-now-what-2apj).
- The [first alpha of React 18 is available](https://github.com/reactwg/react-18/discussions/9) – it's *not* for production use.
- A new Suspense-based [server side rendering architecture](https://github.com/reactwg/react-18/discussions/37).
- A [Transitions API](https://github.com/reactwg/react-18/discussions/41) for keeping an app responsive even during large updates.
- The launch of a [React 18 Working Group](https://github.com/reactwg/react-18) to enable invited members of the community to interact with the React team more directly through [these public discussions](https://github.com/reactwg/react-18/discussions).

The React Core Team
