[`axios` Package Compromised; Malicious Versions Added a Trojan Dependency](https://www.stepsecurity.io/blog/axios-compromised-on-npm-malicious-versions-drop-remote-access-trojan) — [Axios](https://axios-http.com/) is an HTTP library that gets 100M+ downloads a week, largely due to its legacy popularity. An attacker took advantage of that to roll out a version with a malicious dependency including a remote access trojan (though Axios' codebase *itself* was fine). This is big, as even if *you* don’t use Axios, your dependencies might. Here's [how to see if you're affected.](https://www.stepsecurity.io/blog/axios-compromised-on-npm-malicious-versions-drop-remote-access-trojan#am-i-affected)

Ashish Kurmi

💡 **More:** Socket offers [a more accessible breakdown](https://socket.dev/blog/axios-npm-package-compromised). There's also [a GitHub issue](https://github.com/axios/axios/issues/10604) discussing the matter. It's worth considering [pinning your dependencies](https://maxleiter.com/blog/pin-dependencies), preventing post-install scripts from running (can be configured with `npm` but is the default in pnpm and Bun) and/or using cooldowns for dependency updates (using [`minimumReleaseAge`](https://socket.dev/blog/npm-introduces-minimumreleaseage-and-bulk-oidc-configuration) in npm or [pnpm's approach](https://pnpm.io/supply-chain-security)).
