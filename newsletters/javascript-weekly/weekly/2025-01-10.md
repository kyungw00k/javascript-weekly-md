**IN BRIEF:**

- 🥊 The Deno vs Oracle fight over the JavaScript™ trademark continues as Oracle has informed Deno [they're not going to voluntarily relinquish the mark.](https://x.com/deno_land/status/1876728474666217739) Deno's job now is to show how JavaScript has long been used as a generic term and hasn't been controlled by Oracle.
- The Express.js team has posted [about the project's recent revival and its plans to push Express.js forward in 2025.](https://expressjs.com/2025/01/09/rewind-2024-triumphs-and-2025-vision.html)
- ❄️ The 'WinterCG' *Web Interoperable Runtimes Community Group*, an effort to promote standards around runtime interoperability of JavaScript runtimes, [has moved to Ecma International](https://deno.com/blog/wintertc) and is now known as [WinterTC (TC55).](https://wintercg.org/)
- 📺 Node.js creator Ryan Dahl gave [a talk at *GOTO Chicago 2024* about Deno 2](https://www.youtube.com/watch?v=Ak-FYSpW-rA), covering Deno's differences to Node and what [Deno 2.0](https://deno.com/blog/v2.0) (and JSR) has to offer JavaScript developers, complete with live demos.

📄 [Play Tetris in a PDF File](https://th0mas.nl/downloads/pdftris.pdf) — (direct link to PDF)

I'll let you decide if this one is fun or frightening! Whether or not this will work depends on your PDF reader or browser support, but it works with Chrome and Firefox, at least.

The PDF document format supports embedded JavaScript and this experiment uses it to implement a game of Tetris. The developer, Thomas Rinsma, has [used Python to output the PostScript that includes the game's JavaScript.](https://github.com/ThomasRinsma/pdftris/blob/main/gengrid.py) Couple that with the fact many browser PDF renderers are *themselves* implemented in JavaScript (e.g. [PDF.js](https://mozilla.github.io/pdf.js/)) and you have a veritable Matryoshka doll of technologies at play here.
