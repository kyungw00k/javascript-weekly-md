# JavaScript Weekly Issue 607: September 23, 2022

## Top Stories

![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/yq7kkmf1sriyakfhdgso.jpg)

[Tesseract.js 3.0: Pure JS OCR for Over 100 Languages](https://tesseract.projectnaptha.com/) — A port of the C++-based [Tesseract](https://github.com/tesseract-ocr/tesseract) library commonly used for visual text recognition. It can even work upon videos in real time. v3 boasts a big performance boost with 84% lower running time and now uses SIMD where supported. [GitHub repo](https://github.com/naptha/tesseract.js).

Tesseract Team

[A Proposed JS Feature: Extractor Objects](https://github.com/tc39/proposal-extractors) — Entering stage one at the latest TC39 meeting is a proposal for being able to define your own logic (such as data validation or transformation) to run as part of the object destructuring process. A proposal worth reviewing.

Ecma TC39 / Ron Buckton

![](https://copm.s3.amazonaws.com/6984ef48.png)

[Hello World. Caching. Globally. Simplified.](https://www.polyscale.ai/) — PolyScale connects to your current database and executes your queries at the edge, for low global latency. Plug-and-play with Postgres, MySQL, MariaDB, MS SQL. PS: we also invalidate automatically 😉

PolyScale.ai sponsor

['React I Love You, but You’re Bringing Me Down'](https://marmelab.com/blog/2022/09/20/react-i-love-you.html) — An opinionated React thought piece – hurrah 😆 The author gets creative by looking at React's progress in a ‘mix tape’ style with code and memes sprinkled in for good measure. *([Hacker News went deep](https://news.ycombinator.com/item?id=32911492) on this item and spawned some interesting [sub-threads.](https://news.ycombinator.com/item?id=32912681))*
